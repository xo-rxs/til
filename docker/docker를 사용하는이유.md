# Docker를 사용하는 이유

## Docker란?

Docker는 애플리케이션과 실행에 필요한 라이브러리, 설정, 의존성을 하나의 **컨테이너(Container)** 에 담아 어느 환경에서나 동일하게 실행할 수 있도록 하는 컨테이너 가상화 플랫폼이다.

---

## Docker를 사용하는 이유

### 1. 개발 환경의 일관성

개발자마다 운영체제나 프로그램 버전이 달라도 동일한 Docker 이미지를 사용하면 같은 환경에서 애플리케이션을 실행할 수 있다. 이를 통해 환경 차이로 발생하는 오류를 줄일 수 있다.

### 2. 쉬운 배포

애플리케이션과 실행 환경을 하나의 이미지로 만들어 배포하기 때문에 서버에서 별도의 설치 과정 없이 동일한 환경으로 실행할 수 있다. 개발 환경과 운영 환경의 차이를 최소화할 수 있다.

### 3. 독립적인 실행 환경

각 컨테이너는 서로 독립적으로 실행되므로 프로젝트마다 다른 버전의 데이터베이스나 개발 도구를 사용해도 충돌이 발생하지 않는다.

### 4. 빠른 실행과 높은 효율성

Docker는 운영체제의 커널을 공유하는 컨테이너 기술을 사용하기 때문에 가상머신(VM)보다 실행 속도가 빠르고 메모리 사용량이 적다.

### 5. 쉬운 서비스 관리

웹 서버, 데이터베이스, 캐시 서버 등 여러 서비스를 Docker Compose를 이용하여 하나의 설정 파일로 관리할 수 있으며, 한 번의 명령으로 실행 및 종료가 가능하다.

---

## Docker 사용 예시

스트리밍 서비스를 개발하는 경우 다음과 같은 여러 서비스를 동시에 사용한다.

* Spring Boot (백엔드)
* MySQL (데이터베이스)
* Redis (캐시)
* Nginx (웹 서버)
* Media Server (영상 스트리밍)

Docker를 사용하면 이러한 서비스를 하나의 Docker Compose 파일로 관리하여 개발 환경과 운영 환경을 동일하게 유지할 수 있다.
