# TIL - OSI 7계층 모델

## 오늘 배운 것

네트워크 통신을 7개의 추상화 계층으로 나눈 표준 모델인 **OSI 7계층**을 정리했다.

---

## OSI 7계층 정리

| 계층 | 이름 | 전송 단위 | 대표 프로토콜 |
|:---:|------|----------|-------------|
| 7 | 응용 계층 (Application) | 데이터 | HTTP, HTTPS, FTP, DNS, SSH |
| 6 | 표현 계층 (Presentation) | 데이터 | TLS/SSL, JPEG, UTF-8 |
| 5 | 세션 계층 (Session) | 데이터 | NetBIOS, RPC, SIP |
| 4 | 전송 계층 (Transport) | 세그먼트 | TCP, UDP |
| 3 | 네트워크 계층 (Network) | 패킷 | IP, ICMP, ARP, BGP |
| 2 | 데이터 링크 계층 (Data Link) | 프레임 | Ethernet, Wi-Fi, MAC |
| 1 | 물리 계층 (Physical) | 비트 | 케이블, 광섬유, 허브 |

---

## 각 계층 요약

### 7계층 - 응용 계층
- 사용자가 직접 접하는 계층
- 웹 브라우저, 이메일 클라이언트 등이 여기에 해당

### 6계층 - 표현 계층
- 데이터 변환, 암호화, 압축 담당
- TLS 암호화, JPEG/MPEG 인코딩 방식

### 5계층 - 세션 계층
- 통신 세션 시작·유지·종료 담당
- 실제 구현에서는 5~7계층이 합쳐지는 경우가 많음

### 4계층 - 전송 계층
- **포트 번호**로 프로세스 식별
- TCP: 신뢰성 보장 (순서 보장, 재전송)
- UDP: 속도 우선 (빠르지만 비신뢰성)

### 3계층 - 네트워크 계층
- **IP 주소** 기반으로 경로 결정 (라우팅)
- 라우터가 이 계층에서 동작

### 2계층 - 데이터 링크 계층
- **MAC 주소**로 같은 네트워크 내 통신
- 스위치, 이더넷, Wi-Fi가 여기에 해당
- 오류 검출·수정 기능 포함

### 1계층 - 물리 계층
- 비트를 전기/광/무선 신호로 실제 전송
- 케이블, 광섬유, 허브, 리피터

---

## 암기법

```
아파서 누워도 인터넷 돼
응용 → 표현 → 세션 → 전송 → 네트워크 → 데이터링크 → 물리
```

---

## 느낀 점

- 실무에서는 OSI보다 **TCP/IP 4계층** 모델을 더 자주 씀
- 지금 하고 있는 스트리밍 서비스 프로젝트 기준으로 보면:
  - **4계층 (전송)** → WebRTC가 UDP 기반으로 동작
  - **7계층 (응용)** → WHIP, HLS, HTTP API 전부 여기에 해당
- 각 계층이 독립적으로 분리되어 있어서, 특정 계층의 프로토콜을 바꿔도 다른 계층에 영향이 없다는 게 핵심 장점
