# TIL: VM과 Container의 차이

> 가상화 방식의 두 축인 가상 머신(VM)과 컨테이너(Container)의 구조적 차이 

## 한 줄 요약

- **VM**: 하이퍼바이저 위에서 게스트 OS 전체(커널 포함)를 통째로 실행
- **Container**: 호스트 OS의 커널을 공유하면서 프로세스만 격리

## 구조 비교

```
[ VM ]                          [ Container ]

┌──────┬──────┬──────┐          ┌──────┬──────┬──────┐
│ App  │ App  │ App  │          │ App  │ App  │ App  │
├──────┼──────┼──────┤          ├──────┼──────┼──────┤
│Guest │Guest │Guest │          │ Bin/ │ Bin/ │ Bin/ │
│ OS   │ OS   │ OS   │          │ Lib  │ Lib  │ Lib  │
├──────┴──────┴──────┤          ├──────┴──────┴──────┤
│   Hypervisor       │          │  Container Engine  │
├────────────────────┤          ├────────────────────┤
│   Host OS          │          │   Host OS (Kernel) │ ← 공유
├────────────────────┤          ├────────────────────┤
│   Hardware         │          │   Hardware         │
└────────────────────┘          └────────────────────┘
```

## 핵심 차이점

| 항목 | VM | Container |
| --- | --- | --- |
| 가상화 레벨 | 하드웨어 레벨 | OS 레벨 |
| 커널 | 각자 독립된 커널 | 호스트 커널 공유 |
| 크기 | GB 단위 | MB 단위 |
| 시작 속도 | 수십 초 ~ 분 (OS 부팅) | 초 단위 (즉시) |
| 자원 오버헤드 | 큼 (게스트 OS가 자원 점유) | 작음 (커널 공유로 효율적) |
| 격리 강도 | 강함 (커널까지 완전 분리) | 상대적으로 약함 (커널 공유) |
| OS 선택 | 호스트와 다른 OS 가능 | 호스트 커널에 종속 |

## 격리 메커니즘

- **VM**: 하이퍼바이저가 하드웨어를 가상화해 각 VM에 독립된 환경 제공
- **Container**: 리눅스 커널의 두 기능으로 격리 구현
  - **Namespace**: 프로세스, 네트워크, 파일시스템 등의 가시 범위 분리
  - **cgroups**: CPU, 메모리 등 자원 사용량 제한

## 비유로 이해하기

- **VM** = 한 부지에 독립된 집 여러 채를 짓는 것 (배관·전기 전부 따로)
- **Container** = 한 집을 방으로 나눠 쓰는 것 (배관·전기는 공유, 방문만 잠금)

## 언제 무엇을 쓰나

- **Container**: 가볍고 배포가 빨라 마이크로서비스, CI/CD에 적합
- **VM**: 다른 OS를 돌려야 하거나 강한 보안 격리가 필요할 때
- 실무에서는 **VM 위에 컨테이너를 올리는** 하이브리드 구성도 흔함
