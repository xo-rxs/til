# RAG (Retrieval-Augmented Generation)

## 개념

RAG는 LLM이 답변을 생성하기 전에 외부 지식에서 관련 정보를 검색해서 가져오는 기술이다.

LLM 혼자 암기한 지식(파라미터)만으로 답하는 게 아니라, 질문에 맞는 자료를 찾아 펼쳐놓고 보면서 답을 쓰는 **오픈북 시험**과 같다.

## 동작 흐름

### 1. 저장 (인덱싱)
- 문서를 작은 청크(chunk)로 나눔
- 각 청크를 임베딩(벡터)으로 변환
- 벡터 DB에 저장

### 2. 검색 (Retrieval)
- 사용자 질문이 들어오면 질문도 벡터로 변환
- 벡터 DB에서 의미적으로 가장 유사한 청크들을 찾음

### 3. 생성 (Generation)
- 검색된 청크를 프롬프트에 함께 넣어 LLM에게 전달
- LLM이 그 자료를 근거로 답변을 생성

## 왜 쓰는가

- **최신성**: LLM은 학습 시점 이후 정보를 모르는데(knowledge cutoff), RAG로 최신 문서를 붙이면 답할 수 있음
- **환각 감소**: 실제 검색된 근거를 기반으로 답하므로 hallucination을 줄임
- **재학습 불필요**: 모델 자체를 fine-tuning하지 않고도 특정 도메인 전용 데이터를 활용 가능

## 핵심 용어

| 용어 | 설명 |
|------|------|
| 청킹(Chunking) | 문서를 작은 단위로 분할하는 작업 |
| 임베딩(Embedding) | 텍스트를 벡터(숫자 배열)로 변환한 것 |
| 벡터 DB | 임베딩을 저장하고 유사도 검색을 지원하는 DB |
| 유사도 검색 | 질문 벡터와 가까운 청크를 찾는 과정 |
